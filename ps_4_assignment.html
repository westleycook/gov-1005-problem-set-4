For this problem set, were going to use the American Community Survey. From this very helpful website about tidycensus: American Community Survey data differ from decennial Census data in that ACS data are based on an annual sample of approximately 3 million households, rather than a more complete enumeration of the US population. In turn, ACS data points are estimates characterized by a margin of error. Tidycensus will always return the estimate and margin of error together for any requested variables.

Use the get_acs command in tidycensus to get data at the country level for the year 2018. The variable youll need is B19013_001, or median household income.

Mad Libs

For Mad libs 1-3, well give you the code for a simple function. Each function has one simple error. Once you debug the error, you should use the function to solve the mad lib. You only need to make one change to each chunk of code to get it to run. Hint: when debugging code, its often easiest to start by running the code for yourself and seeing what error message you get.

The first function calculates standard deviation of a variable, plus one.

# This function should take a numeric vector (like our column, estimate) and 
# calculate the standard deviation of the variable, then adds 1.

sd_plus_one <- function(var, na.rm = TRUE){
  x<- sd(var, na.rm = na.rm + 1)
  return(x
  }

sd_plus_one(var = income$estimate)

The second function uses stringr to return just the state for a county, state, evaluation.

# This function should take a county and state pair and extracts just the state
# name. We can do this easily because the state name is always after a comma and
# space.

 get_state <- function(countystate){
  countystate 
  str_split_fixed(.,", ", 2) %>% 
  pluck(2) %>% 
  return()
 }

get_state("Treasure County, Montana")
